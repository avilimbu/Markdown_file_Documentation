# Horizontal Rules in Markdown

A **horizontal rule** is used to separate different sections of a Markdown document, making it more organized and easier to read.

## Syntax

You can create a horizontal rule using any of the following:

### Using Hyphens (`---`)

```md
---
```

Output

---

### Using Asterisks (`***`)

```md
***
```

Output

***

### Using Underscores (`___`)

```md
___
```

Output

___

> **Note:** All three methods produce the same result, but `---` is the most commonly used.

## Example

### Markdown

```md
# Introduction

This is the introduction section.

---

# Conclusion

This is the conclusion section.
```

### Output

# Introduction

This is the introduction section.

---

# Conclusion

This is the conclusion section.

## Best Practices

- Use horizontal rules to separate major sections.
- Leave a blank line before and after the horizontal rule.
- Use one style consistently throughout your document.
- Prefer `---` for better readability.

## Summary

Horizontal rules improve the structure of your document by visually separating content. You can create one using `---`, `***`, or `___`.
