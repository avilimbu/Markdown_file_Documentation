# Task Lists in Markdown

Task lists are used to create **checkboxes** for tracking tasks. They are commonly used in GitHub issues, README files, and project documentation.

## Unchecked Task

Use `- [ ]` to create an unchecked checkbox.

### Markdown

```md
- [ ] Learn Markdown
- [ ] Create a README
```

### Output

- [ ] Learn Markdown
- [ ] Create a README

---

## Checked Task

Use `- [x]` to create a checked checkbox.

### Markdown

```md
- [x] Install Git
- [x] Learn Markdown
```

### Output

- [x] Install Git
- [x] Learn Markdown

---

## Example

### Markdown

```md
# Project Tasks

- [x] Create Repository
- [x] Add README
- [ ] Write Documentation
- [ ] Deploy Project
```

### Output

# Project Tasks

- [x] Create Repository
- [x] Add README
- [ ] Write Documentation
- [ ] Deploy Project

---

## Best Practices

- Use task lists to track progress.
- Mark completed tasks with `- [x]`.
- Keep task descriptions short and clear.

---

## Summary

Task lists make it easy to manage and track work using checkboxes.

| Checkbox | Syntax |
|----------|--------|
| Unchecked | `- [ ] Task` |
| Checked | `- [x] Task` |
